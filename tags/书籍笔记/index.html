拜读Bruce大作 —— 《Java编程思想》


第一章 对象导论

对象是一个通过对问题域中的元素进行建模，是一个其封装了以数据成员或属性表示的内部状态和以成员方法与接口表示的行为的抽象概念。

  • 对象始于抽象 -> 抽象始于问题建模【问题域受限于上下文】；
  • 可替代性(substitutability)：某一特定类型的所有对象都可以接收同样的消息；
  • 每个对象都有一个接口；
  • 使用对象开发的好处：每个对象都提供服务，将对象看作“服务的提供者”有助于提高对象的内聚性；可隐藏具体实现；对象复用，即组合对象以实现灵活的代码复用；便于对问题进行表示和交流；可应用设计良好的继承机制；拥有比如多态等高级开发技巧；
  • 继承关系探讨，“是一个” 与 “像是一个”，【书：有时必须在导出类型中添加新的接口元素，这个新的类型仍然可以替代基类，但是这种替代并不完美，因为基类无法访问新添加的方法。这种情况我们可以描述为is-like-a关系。】
  • 多态的可互换对象，【书：为了执行动态绑定，Java使用一小段特殊代码来替代绝对地址调用；我们要编写一段代码，它忽略类型的具体细节，仅仅和基类交互。这段代码和具体类型信息是分离的…把将导出类看做是它的基类的过程称为向上转型(upcasting)…这里要表达的是，如果你是一个Shape，我知道你可以 erase() 和 draw() 你自己，那么去做吧…】
  • 单根继承结构的好处：简化参数传递，保证所有对象都具有某些功能，执行基本操作；使得垃圾回收器的实现变得容易多；为异常处理等系统级操作带来灵活性；
  • 对象的创建和生命期：如何才能知道何时销毁对象呢？垃圾回收器！
  • 异常处理：异常是一种对象；有助于编写出更健壮程序的机制；

第二章 一切都是对象

  • 用引用操纵对象；
  • 对象存储到什么地方：new对象存储在堆（寄存器(处理器内部)、堆栈、堆、常量存储区、非RAM存储区(流对象和持久化对象-即磁盘等)）；
  • 基本类型存储到什么地方：变量直接存储“值”，并置于堆栈中，更加高效；
  • java中的数组，【书：当创建一个数组对象时，实际上就是创建一个引用数组。并且每个引用都会被初始化为一个特定值，该值拥有自己的关键字null】；
  • 永远不需要销毁对象？！
  • static 关键字：static 方法可以创建或使用与其类型相同的被命名对象；因此常常拿来做“牧羊人”的角色，负责看护与其隶属统一类型的实例群；

拜读K.K.90年大作——《失控》

第1章 人造与天生

第1章是开头，写得简洁扼要，里面的一些预想和劝诫至今仍有效：

  • 机器正在生物化；而生物，正在机器化。
  • 由于我们自己创造的这个世界变得过于复杂，…要想保证一切正常运转，我们最终制造出来的环境越机械化，可能越需要生物化。
  • 钟表般的精确逻辑——即机械的逻辑——只能用来建造简单的装置。真正复杂的系统，…都需要一种地道的非技术的逻辑。
  • 关键词：“活系统”; 自然的野性。

对其中一段翻译的感想：自然是一个“文明基因库”（原文：“文化基因库”）。
